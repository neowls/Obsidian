## 개요
기존 월드 상의 X,Y 벡터에 따라 이동하는 캐릭터가 있다.
입력값 또한 상하좌우에 따라 -1 부터 1 까지 정규화된 값으로 들어오니 이를 AddMovementInput 함수에 매개변수로 넘겨주면 이동 로직을 구현할 수 있다.
하지만 만약 캐릭터가 월드 기준의 X,Y 상의 좌표로 이동하는게 아니라 45도 틀어진 각도에서 움직여야 한다면 어떻게 해야할까?

## 본문
일단 기존의 입력값에 따른 이동 코드는 아래와 같다.
FInputActionValue 값을 통해 들어온 값을 통해 이동을 구현하였다.

```C++
{
	FVector2D MovementVector = Value.Get<FVector2d>(); // X, Y 입력 벡터 저장  
	float MovementVectorSize = 1.0f;  
	float MovementVectorSizeSquared = MovementVector.SizeSquared();    // 이동 속도 제곱 연산  
	if(MovementVectorSizeSquared > 1.0f)    // 조이 스틱 사용시 1.0f 아래의 미세값들을 위한 연산  
	{  
	    MovementVector.Normalize(); // 이동 벡터 정규화  
	}  
	  
	else  
	{  
	    MovementVectorSize = FMath::Sqrt(MovementVectorSizeSquared);   // 이동 속도 제곱근 연산  
	}  
	  
	FVector MoveDirection = FVector(MovementVector.X, MovementVector.Y, 0.0f);  // 이동 방향 벡터 연산  
	
	AddMovementInput(MoveDirection, MovementVectorSize);
}
```

여기서 MoveDirection 의 방향을 45(혹은 -45)도 틀어진 방향의 벡터로 **AddMovementInput** 함수에 전달해야한다.
이 때 **RotateVector** 함수를 활용하여 지정된 FRotator 오프셋 값에 따라 입력된 벡터값을 회전 시켜줄 수 있다.

```C++
// 45도 회전 적용
FRotator RotationOffset(0.0f, 45.0f, 0.0f); // Yaw 값 45도 회전
FVector MoveDirection = FVector(MovementVector.X, MovementVector.Y, 0.0f);
MoveDirection = RotationOffset.RotateVector(MoveDirection); // 벡터를 회전 변환
```

그렇다면 이 **RotateVector** 는 어떻게 동작하는걸까?
해당 함수는 내부적으로 **회전 행렬(Rotation Matrix)을 생성**하고, 해당 행렬을 벡터에 적용하여 회전된 벡터를 반환한다.

```C++
template<typename T>  
UE::Math::TVector<T> UE::Math::TRotator<T>::RotateVector(const UE::Math::TVector<T>& V) const  
{  
    return UE::Math::TRotationMatrix<T>(*this).TransformVector( V );  
}
```

```C++
template<typename T>  
FORCEINLINE TVector4<T> TMatrix<T>::TransformVector(const TVector<T>& V) const  
{  
    return TransformFVector4(TVector4<T>(V.X, V.Y, V.Z, 0.0f));  
}
```

```C++
template<typename T>  
FORCEINLINE TVector4<T> TMatrix<T>::TransformFVector4(const TVector4<T>& P) const  
{  
    TVector4<T> Result;  
    TVectorRegisterType<T> VecP = VectorLoadAligned(&P);  
    TVectorRegisterType<T> VecR = VectorTransformVector(VecP, this);  
    VectorStoreAligned(VecR, &Result);  
    return Result;  
}
```

그래서 쿼터니언(Quaternion) 회전 변환 공식을 찾아보았다.

$$
V′=Q×V×Q−1
$$

- $V$는 기존 벡터 (`MoveDirection`)
- $Q$는 회전을 나타내는 쿼터니언 (`FQuat`)
- $Q^{-1}$는 쿼터니언의 역원

이를 45도 회전에 대한 계산으로 바꿔본다면,

$$
R(45^\circ) = \begin{bmatrix} \cos 45^\circ & -\sin 45^\circ & 0 \\ \sin 45^\circ & \cos 45^\circ & 0 \\ 0 & 0 & 1 \end{bmatrix}​​
$$

즉, 원래 벡터 `(X, Y, 0)`에 이 행렬을 곱하면:

$$
X' = X \cos 45^\circ - Y \sin 45$$
$$
Y'=Xsin⁡45∘+Ycos⁡45∘$$
$$
Z' = Z
$$

만약 `MoveDirection = FVector(1, 0, 0)` 라고 가정하면:

$$
X' = 1 \cdot \cos 45^\circ - 0 \cdot \sin 45^\circ = \frac{\sqrt{2}}{2}
$$
$$
Y' = 1 \cdot \sin 45^\circ + 0 \cdot \cos 45^\circ = \frac{\sqrt{2}}{2}​​
$$
따라서, 결과적으로:
`MoveDirection = FVector(0.707f, 0.707f, 0.0f);

원래는 X = 1, Y = 0 이였던 벡터가 X = 0.707, Y = 0.707 로 회전하게 된다.

결론적으로 `FRotator::RotateVector()`는 내부적으로 `FQuat::RotateVector()`를 호출하여 **쿼터니언 회전 변환을 수행**한다.
쿼터니언 회전은 **벡터 외적과 쿼터니언 곱셈을 사용하여 회전 변환을 적용**한다.
Z축을 기준으로 45도 회전하면 **회전 행렬 기반 변환(코사인, 사인 연산)과 동일한 결과**가 나온다.

## 2D 연산

2D 공간에서 점 `(x, y)`을 원점 `(0, 0)`을 기준으로 **θ (세타) 만큼 회전**하면,  
새로운 좌표 `(x', y')`는 다음 공식을 따른다.

$$
x' = x \cos\theta - y \sin\theta
$$
$$
y' = x \sin\theta + y \cos\theta
$$

이걸 코드로 변환하면,

```C++
FVector2D RotateVector2D(const FVector2D& Vec, float AngleDegrees)
{
    // 각도를 라디안으로 변환 (UE5에서는 보통 라디안을 사용)
    float AngleRadians = FMath::DegreesToRadians(AngleDegrees);

    // cos, sin 값 계산
    float CosTheta = FMath::Cos(AngleRadians);
    float SinTheta = FMath::Sin(AngleRadians);

    // 회전 변환 공식 적용
    float X_New = Vec.X * CosTheta - Vec.Y * SinTheta;
    float Y_New = Vec.X * SinTheta + Vec.Y * CosTheta;

    return FVector2D(X_New, Y_New);
}
```
