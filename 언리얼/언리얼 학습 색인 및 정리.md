현재 노트는 단순 이론 정리보다 `이론 -> C++/블루프린트 적용 -> 프로젝트 기능 문서화` 흐름이 강하다.

## 내가 배운 내용 정리

### 1. GAS를 구조적으로 이해했다

아래 개념을 이론 문서와 실전 문서 양쪽에서 반복적으로 다뤘다.

- `ASC`, `GA`, `GE`, `GC`, `GameplayTag`, `AttributeSet`, `Meta Attribute`, `Spec`, `TargetActor`, `AbilityTask`
- `InputID` 기반 입력 처리와 스펙 조회
- `PlayerState`와 `Character` 중 어디에 ASC를 둘지에 대한 구조적 판단
- `CurveTable`과 레벨 기반 수치 확장
- Gameplay Effect 생성 흐름과 컨텍스트/스펙 개념
- 멀티플레이어 예측 개념에 대한 기본 이해

실전 적용 흔적:
- [어빌리티 리팩토링](<프로젝트/ProjectKY/캐릭터/어빌리티 리팩토링.md>) 에서 `GameplayTag -> DataAsset -> Montage/Event` 흐름으로 베이스 어빌리티를 일반화했다.
- [어트리뷰트 세트](<프로젝트/ProjectKY/캐릭터/어트리뷰트 세트.md>) 와 [체력 UI](<프로젝트/ProjectKY/캐릭터/체력 UI.md>) 에서 GAS와 UI를 연결했다.

판단:
- `GAS`는 현재 가장 강한 축 중 하나다.
- 특히 "개념 암기"보다 "프로젝트 구조에 맞게 재배치하는 능력"이 보인다.

## 결론

지금까지의 학습은 `언리얼 전반`을 넓게 얕게 훑은 상태가 아니라, `GAS + 액션 캐릭터 + 애니메이션 + AI + UI` 라는 명확한 축 위에 상당히 강하게 쌓여 있다.

반대로 `엔진 레벨 운영`, `월드 관리`, `네트워크`, `패키징`, `PCG`, `오디오`, `온라인`, `자동화`는 아직 본격적으로 들어가지 않았다.

즉, 현재 상태는:

- **게임플레이 구현 역량은 강함**
- **엔진 운영/대규모 프로젝트 운영 역량은 앞으로 채워야 함**

다음 단계는 "기능 구현"에서 한 단계 올라가 `UE 5.7 엔진 구조`, `프로젝트 운영`, `확장 가능한 시스템`, `빌드/패키징/디버깅/프로파일링` 쪽으로 넓히는 것이다.
