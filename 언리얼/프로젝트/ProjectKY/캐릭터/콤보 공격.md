## 개요
근접 공격은 콤보의 시작(Start)-진행(Playing)-끝(End)으로 이루어지며, 상태와 입력에 따라 콤보 진행이 달라야한다.
- 입력을 통해서 행동이 유발된다.
- 현재 공격을 할 수 있는지 상태를 확인한다.
- 현재 지상인지 공중인지 상태를 확인한다.
- 공중이라면 잠시 체공 상태를 둔다.
- 전방에 공격을 가한다.

# 콤보
- 콤보는 예약 입력으로 진행된다.
- 콤보 진행중 입력이 들어온다면 다음 콤보가 가능한 타이밍에 바로 실행한다.
- 이미 콤보 예약 입력이 들어와있다면 대쉬를 제외한 다른 입력은 무시한다.

# 콤보 진행 중 입력에 따른 진행

|           | 콤보 시작 구간 | 콤보 예약 입력 가능 구간 |      콤보 예약 입력 확인 구간      |
| :-------: | :------: | :------------: | :----------------------: |
| **기본 공격** | 동작하지 않음  |    기본 콤보 예약    |     기본 콤보 진행하며 입력 무시     |
|  **점프**   | 동작하지 않음  |   에어리얼 공격 예약   |        에어리얼 공격 실행        |
|  **이동**   | 동작하지 않음  |    동작하지 않음     |     애니메이션 중단 후 바로 이동     |
|  **대쉬**   | 동작하지 않음  |     콤보 종료      |                          |
|  **미입력**  | 동작하지 않음  |    동작하지 않음     | 콤보 애니메이션은 진행되며 대기 상태로 전환 |

## 구현 과정

### GA
ActivateAbility 를 통해 상태에 따른 콤보를 시작한다.
애니메이션 몽타주가 진행되면서 노티파이 스테이트에 의해 입력 가능 태그가 전달된다.
미리 다음 콤보를 위한 입력을 진행했다면 입력 스팸을 막기 위한 bool 값(HasNextComboInput)을 활성화한다.
입력 가능 태그가 끝나는 것을 감지하기 위해 `UAbilityTask_WaitGameplayTagRemoved` AT를 생성하고 콤보 함수를 콜백함수로 등록한다.
또한 히트 이벤트가 감지되면 타겟에 대한 데이터를 캐릭터에서 공격 어빌리티에게 전달한다.

```C++
  
void UKYGA_ComboAttack::InputPressed(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo,  
    const FGameplayAbilityActivationInfo ActivationInfo)  
{  
    Super::InputPressed(Handle, ActorInfo, ActivationInfo);  
    UAbilitySystemComponent* SourceASC = GetAbilitySystemComponentFromActorInfo_Checked();  
    if (SourceASC->GetOwnedGameplayTags().HasTagExact(KYTAG_EVENT_COMBO_AVAILABLE) && !HasNextComboInput)  
    {  
       HasNextComboInput = true;  
       UAbilityTask_WaitGameplayTagRemoved* ComboValidTask = UAbilityTask_WaitGameplayTagRemoved::WaitGameplayTagRemove(this, KYTAG_EVENT_COMBO_AVAILABLE);  
       ComboValidTask->Removed.AddDynamic(this, &UKYGA_ComboAttack::InputCallback);  
       ComboValidTask->ReadyForActivation();  
    }  
}  
  
  
void UKYGA_ComboAttack::InputCallback_Implementation()  
{  
    StartNextCombo();  
}  
  
FName UKYGA_ComboAttack::GetNextSection()  
{  
    CurrentAttackIndex = FMath::Clamp(CurrentAttackIndex + 1, 0, MaxCombo);  
    FName NextSection = *FString::Printf(TEXT("Attack%d"), CurrentAttackIndex);  
      
    return NextSection;  
}  
  
void UKYGA_ComboAttack::StartNextCombo()  
{  
    if (HasNextComboInput)  
    {  
       MontageJumpToSection(GetNextSection());  
       HasNextComboInput = false;  
    }  
}
```

- ActivateAbility
	- 콤보 인덱스를 초기화한다.
	- 콤보 입력 bool 값을 초기화한다.

- InputPressed
	- 현재 근접 공격이 가능한 상태인지 태그(KYTAG_EVENT_COMBO_AVAILABLE)를 확인한다.
	- 또한 이미 입력이 된상태인지 확인한다.
	- 여기서 입력 가능 태그가 끝나는 것을 감지하기 위해 `UAbilityTask_WaitGameplayTagRemoved` AT를 생성하고 다음 콤보 실행 함수를 콜백함수로 등록한다.

- CancelAbility
	- 현재 진행중인 몽타주를 중단한다.

### 공중 공격
![[Pasted image 20241213171756.png]]

공중에서 공격시에는 태그에 따라 체공 상태 공격을 진행한다.
이는 CharacterMovementComponent 를 확장하여 현재 캐릭터가 체공 상태 태그(KYTAG_CHARACTER_ISGLIDING)를 가지고 있다면 체공 상태로 변경한다.
체공 상태는 아래로 떨어지는 속력값을 클램핑하여 제어하였다.

```C++
void UKYCharacterMovementComponent::UpdateCharacterStateAfterMovement(float DeltaSeconds)  
{  
    Super::UpdateCharacterStateAfterMovement(DeltaSeconds);  
  
    AKYCharacterBase* CharacterBase = Cast<AKYCharacterBase>(GetOwner());  
      
	if(CharacterBase)  
	{  
	    if (IsFalling() && !CharacterBase->GetAbilitySystemComponent()->HasMatchingGameplayTag(KYTAG_CHARACTER_ISFALLING))  
	    {  
	       CharacterBase->GetAbilitySystemComponent()->AddLooseGameplayTag(KYTAG_CHARACTER_ISFALLING);  
	    }  
	      
	    else if (!IsFalling() && CharacterBase->GetAbilitySystemComponent()->HasMatchingGameplayTag(KYTAG_CHARACTER_ISFALLING))  
	    {  
	       CharacterBase->GetAbilitySystemComponent()->RemoveLooseGameplayTag(KYTAG_CHARACTER_ISFALLING);  
	    }  
	      
	    if (CharacterBase->GetAbilitySystemComponent()->HasMatchingGameplayTag(KYTAG_CHARACTER_ISGLIDING))  
	    {  
	       Velocity.Z = FMath::Clamp(Velocity.Z, -MaxFallingSpeed, 1200.0f);  
	    }  
}
}
```
