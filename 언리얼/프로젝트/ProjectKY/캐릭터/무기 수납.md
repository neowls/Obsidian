## 개요
캐릭터의 전투(컴뱃)상태에 따라 무기를 수납하고 꺼내는 기능을 구현한다.

## 구현
```C++
// KYGameplayAbility
UPROPERTY(EditDefaultsOnly, Category="Activation")  
bool bIsCombatAbility = false;

void UpdateCombatStateTag(const FGameplayAbilityActorInfo* ActorInfo);

//KYCharacterBase
UFUNCTION(BlueprintCallable)  
FORCEINLINE FTimerHandle& GetCurrentCombatTimerHandle() { return CombatTagTimerHandle; }

UPROPERTY()  
FTimerHandle CombatTagTimerHandle;
```

베이스 어빌리티 클래스 멤버 변수로 해당 어빌리티가 컴뱃(전투 관련) 어빌리티인지 체크할 수 있는 부울 변수를 두었다.
또한 현재 전투 상태임을 업데이트 할 수 있는 함수를 선언해주었다.
캐릭터는 전투 상태가 지속되는 시간을 관리하는 타이머 핸들을 가지고 있도록 하였다.

```C++
void UKYGameplayAbility::ActivateAbility(const FGameplayAbilitySpecHandle Handle,  
    const FGameplayAbilityActorInfo* ActorInfo, const FGameplayAbilityActivationInfo ActivationInfo,  
    const FGameplayEventData* TriggerEventData)  
{  
    Super::ActivateAbility(Handle, ActorInfo, ActivationInfo, TriggerEventData);  
  
    AKYCharacterBase* Character = Cast<AKYCharacterBase>(ActorInfo->AvatarActor);  
    UAbilitySystemComponent* ASC = Character->GetAbilitySystemComponent();  
    if (!ASC) return;  
      
    if (bIsCombatAbility)  
    {  
       if (ASC->HasMatchingGameplayTag(UKYGameplayTags::CharacterState.IsCombat)) GetWorld()->GetTimerManager().ClearTimer(Character->GetCurrentCombatTimerHandle());  
       else  
       {  
          ASC->AddLooseGameplayTag(UKYGameplayTags::CharacterState.IsCombat);  
       }  
    }  
}

void UKYGameplayAbility::UpdateCombatStateTag(const FGameplayAbilityActorInfo* ActorInfo)  
{  
    AKYCharacterBase* Character = Cast<AKYCharacterBase>(ActorInfo->AvatarActor);  
    UAbilitySystemComponent* ASC = Character->GetAbilitySystemComponent();  
    if (!ASC) return;  
      
    if (ASC->HasMatchingGameplayTag(UKYGameplayTags::CharacterState.IsCombat))  
    {  
       GetWorld()->GetTimerManager().SetTimer(Character->GetCurrentCombatTimerHandle(), [ASC]()  
       {  
          if (ASC)  
          {  
             ASC->RemoveLooseGameplayTag(UKYGameplayTags::CharacterState.IsCombat);  
          }  
       }, 5.0f, false);  
    }  
}
```

어빌리티가 실행이 되면 컴뱃 어빌리티인지 확인하고, 이에 따라 컴뱃 태그를 부착한다.
만약 이미 부착된 상태라면 다른 어빌리티에 의해 전투 상태가 된 상태이므로, 캐릭터의 타이머를 초기화하고, 아니라면 컴뱃 태그를 붙여준다.
이렇게 하여 앞서 다른 컴뱃 어빌리티와의 시간차에 따른 타이머 종료가 겹치는 현상을 방지할 수 있었다.

```C++
void UKYGameplayAbility::EndAbility(const FGameplayAbilitySpecHandle Handle, const FGameplayAbilityActorInfo* ActorInfo,  
    const FGameplayAbilityActivationInfo ActivationInfo, bool bReplicateEndAbility, bool bWasCancelled)  
{  
    Super::EndAbility(Handle, ActorInfo, ActivationInfo, bReplicateEndAbility, bWasCancelled);  
    if (bIsCombatAbility) UpdateCombatStateTag(ActorInfo);  
}
```

그러므로 태그가 지워지는 타이밍은 항상 어빌리티가 끝날 때 함수 호출을 통해 타이머를 동작시켜서 진행한다.
대신, 위와 같이 다른 어빌리티에 의해 타이머가 초기화 되어 태그가 지워지지 않고 유지될 수 있으며, 이어지는 어빌리티의 종료 시점에 의해 지워지게 된다.

```C++
// Base
void AKYCharacterBase::OnCombatState(const FGameplayTag GameplayTag, int32 Count)  
{  
    bIsCombat = Count > 0;  
    if (bIsCombat)  
    {  
       UpdateWeaponSocket();  
       if (CurrentWeaponType == EWeaponType::DualBlade)  
       {  
          UpdateWeaponSocket(false, TEXT("Weapon_L"));  
       }  
    }  
}

void AKYCharacterBase::UpdateWeaponSocket(bool IsRight, FName TargetSocket)  
{  
    if (IsRight) WeaponCompRight->AttachToComponent(GetMesh(), FAttachmentTransformRules::SnapToTargetNotIncludingScale, TargetSocket);  
    else  
    {  
       WeaponCompLeft->AttachToComponent(GetMesh(), FAttachmentTransformRules::SnapToTargetNotIncludingScale, TargetSocket);  
    }  
}

// Player
void AKYCharacterPlayer::OnCombatState(const FGameplayTag GameplayTag, int32 Count)  
{  
    Super::OnCombatState(GameplayTag, Count);  
    if (!bIsCombat)  
    {       PlayAnimMontage(GetAnimMontageData(UKYGameplayTags::Data.Ability_Common_Combat).Montage, 1.0f, TEXT("NonCombat"));  
    }  
}


```

전투 상태의 감지는 태그로 확인하기 때문에, 현재 태그가 등록 해제되는 것을 감지하는 AbilitySystemComponent 의 RegisterGameplayTagEvent 델리게이트를 활용하였다.
전투 상태가 되면 즉각적으로 현재 무기를 소켓에 부착하고, DualBlade 타입의 무기라면 좌측또한 부착한다.

비전투 상태가 되면 현재 무기를 집어넣는 몽타주를 재생하며, 몽타주에서는 수납 타이밍에 맞춰 각 무기를 다시 타입에 따라 지정된 소켓에 수납하도록 노티파이를 지정하였다.
노티파이에서는 소켓 이름과 주무기(오른쪽)인지 설정한 값에 따라 상단의 Update Weapon Socket 함수를 실행한다.
