## 개요
[[게임플레이 큐(GC)]] 와 다이나믹 머터리얼 인스턴스를 활용하여 구현한다.

## 구현
### 게임플레이 큐(Gameplay Cue)
우선 피격시 나타나는 일시적인 이펙트이므로 게임플레이 큐 스태틱을 상속받아 제작하였다.

![[HitEffect_0.png]]

이후 Execute 함수를 오버라이드 하여 내부에서 나이아가라 시스템을 타겟 로케이션에 스폰시켜주었다.

![[HitEffect_1.png]]

또한 큐가 발동될 조건 태그를 설정하였다.

![[HitEffect_2.png]]

Gameplay Effect 에서 게임플레이 큐 발동을 시킬 수 있도록 되어있으며, 이곳에 위와 같은 태그를 지정하면 해당 게임 플레이 이펙트가 실행될 때 큐도 실행된다.

![[HitEffect_3.png]]

### 다이나믹 머터리얼 인스턴스(Dynamic Material Instance)
런타임중 머터리얼에 값을 전달하여 효과를 나타낼 때 주로 사용된다.
우선 피격시 테두리부터 중앙으로 옅게 퍼진 빛이 순간적으로 번쩍이는 효과를 만들고자 했다.

기존에 사용되고 있는 적 머터리얼을 수정하려 했는데, 평소 보았던 머터리얼 형태가 아닌 머터리얼 어트리뷰트를 사용하고 있었다.
조금 둘러보아 레이어 블렌드라는 것을 통해 머터리얼을 구성하고 있다고 판단하여, 필요한건 마지막 Emissive 값의 수정이니 적절한 레이어 블렌드를 찾아보았다.

![[HitFlash_0.png]]
들어온 베이스 머터리얼값에 이미시브만 더해져서 나가는 형태의 레이어 블렌드 함수다.
여기에 Fresnel 에 지정된 컬러를 곱하고, Emissive 강도를 곱하여 나온 값을 넣어주었다.
또한 다이나믹 인스턴스를 통해 값을 전달받기 위해 색상(HitFlashColor)과 번쩍이는 강도(HitFlashAmount)를 파라미터로 두었다.

> [!info] 프레넬(Fresnel)
> Fresnel 은 관찰자가 바라보는 각도에 따라 반사되는 빛의 세기가 달라지는 현상을 설명하는 데 사용되는 용어이다.
> 언리얼에서는 Fresnel 머티리얼 표현식 노드는 표면 노멀과 카메라까지의 방향의 내적을 기반으로 감쇠 계산을 한다.
> 
> - 지수 입력(ExponentIn)은 프레넬 이펙트가 감쇠되는 정도를 뜻하며,
> - 기본 리플렉션 굴절 입력(BaseReflectFractionIn)은 스페큘러 리플렉션의 굴절율을 뜻한다.
> - 노멀(Normal)은 노멀 맵이나 벡터3 입력을 통해 렌더링 방식에 영향을 끼칠 수 있다.
> 
> ![[Fresnel.png]]

이후 해당 베이스 머터리얼을 토대로 머터리얼 인스턴스를 생성하고, 적 캐릭터 메쉬에 등록해주었다.
또한 캐릭터가 생성될 시 다이나믹 머터리얼 인스턴스를 생성하고, 이를 변수로 저장하게 해주었다.

![[HitFlash_1.png]]

피격시 나타나는 효과이여야 하므로, 이를 캐릭터 베이스 코드에서 DamageTaken 함수 호출시 같이 호출되도록 **BlueprintImplementableEvent** 형태로 함수를 생성해 주었다.

```C++
UFUNCTION(BlueprintImplementableEvent)  
void UpdateHitFlash();
```
``
```C++
void AKYCharacterBase::DamageTaken(AActor* DamageInstigator, AActor* DamageCauser, const FGameplayTagContainer& GameplayTagContainer, float Damage)  
{  
    FGameplayTagContainer FilteredGameplayTags(FGameplayTag(KYTAG_CHARACTER_ATTACK));  
      
    for (auto& InGameplayTag : GameplayTagContainer.Filter(FilteredGameplayTags))  
    {  
       FGameplayEventData EventData;  
       EventData.Instigator = DamageCauser;  
       EventData.EventTag = InGameplayTag;  
       EventData.EventMagnitude = Damage;  
  
       KY_LOG(LogKY, Log, TEXT("InGameplayTag Before Send : %s"), *InGameplayTag.GetTagName().ToString());  
       UpdateHitFlash();  // 피격 플래쉬 이펙트 발동
       UAbilitySystemBlueprintLibrary::SendGameplayEventToActor(this, InGameplayTag, EventData);  
    }  
}
```

![[HitFlash_2.png]]

블루프린트에서 Event Update Hit Flash 이벤트를 상속받아 생성해주고, 타임라인을 통해 이전에 만들어둔 다이나믹 머터리얼 인스턴스에 접근해 강도를 조절하여 구현하였다.
