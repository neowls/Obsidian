# 객체지향(OOP)
프로그래밍에서 필요한 데이터를 추상화시켜 상태와 행위를 가진 객체를 만들고 그 객체들 간의 유기적인 상호작용을 통해 로직을 구성하는 프로그래밍 방법

## 장점
- 코드 재사용성 용이
- 유지 보수 용이
- 대형 프로젝트에 적합

## 단점
- 처리속도가 상대적으로 느림
- 객체가 많으면 용량이 커질 수 있음

## 4대 특징
- 캡슐화
	기능과 특성을 클래스라는 캡슐에 분류해서 넣음으로서 코드 재활용과 정보 은닉

- 상속
	부모클래스의 속성과 기능을 그대로 이어받아 재사용할 수 있게 하는 것

- 추상화
	공통의 속성이나 기능을 묶고 이름을 붙여 클래스를 설계하는 것

- 다형성
	하나의 변수명 함수명 등이 상황에 따라 다른 의미(용도)로 해석될 수 있는 것
	오버라이딩 - 부모 클래스의 메서드와 같은 이름, 매개변수를 재정의 하는 것
	오버로딩 - 같은 이름의 함수를 여러개 정의하고, 매개변수의 타입과 개수를 달리하여 상황에 따라 다르게 호출할 수 있게 하는 것

## 5대 원칙(SOLID)
- 단일 책임 원칙 (Single responsibility principle)
	한 클래스는 하나의 책임만 가져야 한다.

- 개방-폐쇄 원칙 (Open/closed principle)
	소프트웨어 요소는 확장에는 열려있으나 변경에는 닫혀 있어야 한다.

- 리스코프 치환 원칙 (Liskov substitution principle)
	프로그램의 객체는 프로그램의 정확성을 깨뜨리지 않으면서 하위 타입의 인스턴스로 바꿀 수 있어야 한다.

- 인터페이스 분리 원칙 (Interface segregation principle)
	특정 클라이언트를 위한 인터페이스 여러 개가 범용 인터페이스 하나보다 낫다.

- 의존관계 역전 원칙 (Dependency inversion principle)
	추상화에 의존하되, 구체화에 의존하면 안된다.

# 인스턴스화
클래스에서 정의한 것을 토대로 실제 메모리상에 할당하는 것

# 디자인 패턴
- 싱글톤(Singleton)
	인스턴스의 포인터를 전역으로 접근하면서 오직 하나의 인스턴스를 생성하는 것을 보장하는 패턴

- 옵저버(Observer)
	어떤 문제가 발생할 때 주체 클래스에서 갱신된 정보를 관찰자 클래스에 전달하여 처리하는 패턴

- 상태(State)
	어떤 객체가 상태를 가진 클래스 또는 클래스에 있는 Enum 타입에 클래스 변화 또는 식별자의 나열로 정의되어진 함수에 변화를 어떤 객체가 받아 변화하는 패턴

# malloc / free 와 new / delete 차이
malloc / free 는 라이브러리가 제공하는 함수이지만 new / delete 는 언어가 제공하는 연산자이다.

- malloc
	- 필요한 만큼 할당 후 void* 반환해서 SizeOf() 함수가 필요하다.
	- 메모리 할당의 역할일 뿐 초기화는 못한다.

- new
	- C++ 기반의 메모리 동적 할당 연산자이다.
	- 해당 타입의 주소값을 리턴해서 메모리 사이즈 측정 함수가 필요하지 않다.
	- 해당 생성자를 통해서 값 초기화가 가능하다.

- free
	- 라이브러리 함수이다.
	- 해당 메모리 해제외에는 기능이 없다.

- delete
	- 언어가 제공하는 연산자이다.
	- 메모리 해제 및 소멸자를 호출한다.

# C++ struct / class 차이점
기본 접근 지정자의 차이로서 struct 는 public, class 는 private 로서 둘 다 참조 타입이다.

# Call by value / Call by reference 차이점
함수 호출방식의 차이점으로서 value 는 값에 의한 호출이고, reference 는 참조에 의한 호출이다.

# 포인터란
메모리의 주소값을 저장하는 변수이며, 동적 할당이 가능하다.
또한 배열과 구조체 등의 복잡한 자료 구조와 함수에 쉽게 접근할 수 있도록 해준다.

# 포인터와 참조자
포인터는 가리키는 주소값을 자유롭게 변경할 수 있지만, 참조자는 선언 이후 주소값을 변경하는 것이 불가능하다.
포인터는 null 값이 있을 수 있지만, 참조자는 null값이 없다. (선언과 동시에 주소를 지정해주어야 함)
포인터는 주소값을 입력받고 참조자는 변수를 입력받는다.

# 단배열과 이중배열 속도차이
옛날에는 단배열의 속도가 더 빨랐지만, 컴파일러의 발전으로 C++ 14 이상에선 서로 속도가 비슷하다. (단배열 100 사이즈와 이중배열 10x10 사이즈)

# 가상 함수와 가상 테이블
오버라이딩 대상이 되는 함수 생성을 위해 사용된다.
순수 가상함수의 경우는 자식클래스에게 오버라이딩을 강제시키고 싶을 때 사용된다.
가상 함수는 가상 테이블 내에 클래스명, 함수 메모리 주소인 <Key, Value> 형태로 저장된다.
함수가 호출되면 가상 테이블을 통해 적절한 함수를 확인하여 호출해준다.

# 업캐스팅 다운캐스팅
- 업캐스팅
	부모 클래스의 포인터로 자식 클래스 객체를 가리키는 것

- 다운캐스팅
	자식 클래스의 포인터로 부모 클래스의 객체를 가리키는 것(다이내믹 캐스트 필요)

# 다이내믹 캐스트(Dynamic_cast)
RTTI(Run-time type information) 실행 도중 객체의 데이터형을 결정하는 표준 방법을 제공한다.

- 내부 동작
	- 대상의 상속 관계를 판단 후에 관계가 확인된다면 실행한다.(대입 관계 확인 후 실행 아니면 nullptr)
	- 전부 검색을 하기 때문에 느리다.
	- 컴파일 단계가 끝난후, 런타임 단계 때 하기 때문에 안정적이다.

# 컴파일 작동 순서
전처리 -> 컴파일 -> 어셈블 -> 링크

1) 전처리(preprocess)
	- 사전 처리, 소스 코드의 Main 이 실행되기 전에 사전 준비를 하는 과정
	- 컴파일러가 작업하기 좋도록 소스를 재구성한다. (#include, #.define 등)
	- .i 파일을 생성한다.

2) 컴파일(Compile)
	- 전처리된 소스를 컴파일해서 어셈블리어로 변환한다.
	- .i 파일을 .s 파일로 변환 생성한다.

3) 어셈블(Assemble)
	- 어셈블리어를 기계어로 변환한다.
	- .s 파일을 .o 파일로 변환 생성한다.

4) 링크(Link)
	- 만들어진 기계어를 사용하여 파일을 실행한다.
	- 파일의 관계가 연결되고, 필요로 하는 라이브러리도 함께 코드화하는 작업을 실행한다.
	- .o 파일을 실행 파일로 변환한다.

# Abstract 와 Interface
둘 다 new 로 할당 불가하며, 무조건 상속으로 사용해야 한다.

Abstract 는 순수 가상함수가 하나 이상 포함되어 있는 클래스이다.
Interface 는 순수 가상함수로만 이루어진 추상 클래스이다.

# 얕은 복사, 깊은 복사
## 얕은 복사(Shallow Copy)
- 기본 복사 생성자를 사용한 복사이며, 복사할 때 참조 타입의 경우 참조값을 복사하기에 같은 메모리 공간을 공유하게 된다.
- 복사가 발생했을 때 대상이 1개인데 대상에 접근하는 포인터가 2개인 경우 얕은 복사(shallow copy)라고 한다.

## 깊은 복사(Deep Copy)
- 복사가 발생했을 때 대상 각각에 접근하는 포인터가 1개씩 있을 경우 깊은 복사(deep copy)라고 한다.
- 복사 생성자를 재정의해서 참조타입의 참조값이 가리키는 메모리의 값을 복사해온다.
- 객체가 참조 타입의 멤버를 포함할 경우 참조된 객체 자체가 복사되는 것을 깊은 복사라고 한다.

# 복사 생성자, 이동 생성자
## 복사 생성자
객체의 복사본을 생성할 때 호출되는 생성자
- 디폴트 복사 생성자와 명시적 복사 생성자
	- 복사 생성자를 정의하지 않으면 컴파일러가 복사 생성자를 생성하여 호출하며 이것이 디폴트 복사 생성자이다.
	- 정의하면 컴파일러가 복사 생성자를 생성하지 않는데 이런 정의한 복사 생성자가 명시적 복사 생성자이다.

- 복사 생성자의 호출 시점 3가지
	- 먼저 생성한 객체를 나중에 생성하는 객체의 인자로 전달하는 경우
	- 함수의 인자로 객체가 전달되는 경우
	- 함수의 반환값으로 객체가 반환되는 경우

- 복사 생성자가 인자로 레퍼런스 타입을 받는 이유
	- 생성자는 스택 영역에 할당되는데 이때 지역 변수를 할당하고 인자로 넘겨주는 값을 복사 받을 때 다시 그것을 복사할 복사 생성자를 호출해야하는 문제가 발생해서 무한루프가 걸리게된다.
	- 레퍼런스 타입의 인자를 받을 때 그것을 받을 복사생성자가 필요하고, 그 복사 생성자를 받을 복사 생성자가 필요하고 그것을 반복한다.

## 이동 생성자
다른 객체 멤버 변수들의 소유권을 가져오는 생성자
- 복사 생성자와 달리 메모리를 재할당하지 않기에 복사 생성자보다 빠르다.
- 얕은 복사와 비슷하다.
- 이동 대입 연산자 또한 같은 개념으로서 다른 개체의 멤버 변수들의 소유권을 가져온다.

# 동기 / 비동기
## 동기
- 함수를 호출 했을때 이 **함수의 결과값을 직접 받는 것**을 동기라고 한다.
- A 작업을 시작시키고 A 작업이 끝날 때까지 기다렸다가 B 작업을 시작한다.
- 요청한 결과값이 return 값과 동일하다.

## 비동기
- 동기와 달리 함수를 호출 했을때 이 **함수의 결과값을 간접적으로 받는 것**을 비동기라고 한다.
- A 작업을 시작시키고 A 작업을 완료시키지 않고 B 작업을 시작할 수 있다.
- 요청한 결과값이 return 값과 다를 수 있다.

# public, protected 상속
## public
- 부모 클래스에서 상속받은 변수나 함수 등의 접근 지정자를 그대로 상속한다.
- 모든 접근 지정자에 변화가 없다.

## protected
- 부모 클래스에서 상속 받은 변수나 함수 등의 접근 지정자를 최소 protected 로 만든다.
- public 을 protected 로 변경하며, 이외의 protected 나 private 는 변경이 없다.

# 스택(Stack)
- 나중에 들어온게 먼저 나가는 후입선출 구조(LIFO)
- 함수의 호출과 관계되는 지역 변수와 매개 변수가 저장된다.
- 스택의 가장위에서만 자료의 삽입과 삭제가 가능하다
- 비용이 적고 처리속도가 매우 빠르다

# Stack Overflow
- 스택 메모리가 지정된 영역을 벗어난 경우
- 지역 변수가 너무 크거나 재귀함수가 무한정 호출 될 경우 발생한다.

# 힙(Heap)
- 먼저 들어온게 먼저 나가는 선입선출 구조(FIFO)
- 사용자가 동적 할당한게 저장된다.
- 힙은 낮은 주소에서 높은 주소의 방향으로 할당된다.
- 비용이 많고 상대적으로 느리다.
- 힙 영역이 지정된 영역을 벗어난 것을 힙 오버 플로우라고 한다.

# 맵(Map)과 해시맵(Hash Map)
## 맵(Map)
- 맵은 이진 탐색 트리(BST)를 개량한 레드블랙트리(RBT)를 사용해서 구현한다.
- 맵은 값을 저장을 하면서 BST에 의해서 정렬도 진행을 한다.
- 내부 탐색은 O(LogN)의 시간복잡도를 가지며 노드의 삽입 삭제 또한 O(LogN)이다.

## 해시맵(Hash Map)
- unordered_map 이라고도 불리며 해시 테이블(Hash Table)을 사용해서 구현한다.
- 정렬없이 저장만 진행한다.
- 내부 탐색은 O(1)의 시간 복잡도를 가진다.

# 배열과 벡터의 차이점
- 배열은 초기화 이후 크기를 조정할 수 없다.
- 벡터는 다량의 데이터 검색이 상대적으로 느리다.
- 배열은 인덱스 기반의 데이터 저장 컨테이너, 벡터는 순차 데이터 저장 컨테이너 이다.

# 스마트 포인터(Smart Pointer)
메모리 누수를 방지하기 위해 사용되며, 포인터처럼 사용하는 클래스 템플릿으로 메모리를 자동으로 해제해준다.

1) **shared_ptr**
	어떤 하나의 객체를 잠조하는 스마트 포인터의 개수를 참조하는 스마트 포인터이다.

2) **unique_ptr**
	하나의 스마트포인터만이 객체를 가리킬 수 있도록한다.

3) **weak_ptr**
	하나 이상의 shared_ptr 이 가리키는 객체를 참조할 수 있지만, reference count를 늘리지 않는 스마트 포인터이다.

# 메모리 구조, 단편화
## 메모리 구조
- 프로그램을 실행할 때 운영체제에서 메모리에 공간을 할당하는데 그 공간을 4가지로 분류한다.

| Code  | 코드 영역                |
| ----- | -------------------- |
| Data  | 데이터, 전역, 정적 변수       |
| Stack | 지역 변수, 매개 변수, 컴파일 결정 |
| Heap  | 사용자 동적할당, 런타임 결정     |

## 단편화
- 사용 가능한 메모리가 충분히 존재하지만 할당이 불가능한 상태이다.
- 방지하기 위해서는 페이징 기법, 세그멘테이션 기법, 메모리 풀을 사용할 수 있다.

# TCP/UDP
## TCP
- 연결형 서비스 가상 회성 방식 제공
- 데이터의 경계를 구분하지 않음
- 데이터의 전송 순서를 보장함
- UDP보다 전송속도가 느림
- 신뢰성있는 데이터 전송 가능
## UDP
- 비연결형 서비스로 데이터그램 방식을 제공
- 데이터의 경계를 구분함
- 정보를 주고 받을때 정보를 보내거나 받는다는 신호절차를 거치지 않음
- TCP보다 전송속도가 빠름
- 신뢰성 없는 데이터를 전송함

## 결론
- TCP는 UDP보다 더 신뢰성 높은 데이터 전송이 가능
- 대신 UDP는 TCP보다 속도가 빠르며 네트워크 부하가 적다는 장점이 있음
